Online dating is becoming more popular with over 50 million Americans having tried it at some point. Dating apps tend to have negative reputation and not be taken seriously. One problem with dating apps is that a lot of users are complacent to manually enter the preferences making it hard to suggest good matches. The app will learn about the user's swiping patterns using image recognition.

I am using Tensor Flow’s ImageNet library to classify the images, which does not need a powerful GPU. The code can detect multiple objects in a single image by slicing the image into a grid. ImageNet has 1000 classifiers and can identify objects such as sunglasses or cars, locations such as cliffs or beaches, and animals such as dogs or cats. As the user starts browsing profiles, the code will start learning his or her preferences based on the identified objects in the profiles they like. For instance, the code will recognize that the user is inclined to like pets, or that they like sports, or simply like people who spend a lot of time outdoors.

My classification system contains 3 main classes used to build a profile for the user. The first class is location, which determines if the user prefers the indoors or outdoors, and what kind of location they prefer. The second class is pets, which will gauge how much they like pets, then identify their favorite pet whether it’s dogs or cats down to the breed. The third class is sports, which keeps track of the physical activities they like.

For demonstration, I classified objects in 5000 images of men, and 6000 images of women scraped from the popular dating app Tinder. The results are stored in a .csv file separately to allow for faster processing in the website. The code used for processing and parsing the results can be found in the folder /image_processing.
